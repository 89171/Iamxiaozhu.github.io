Babel的简单使用

Babel安装

1.安装babel: npm install –save-dev babel-cli
2.安装babel-preset-es2015预设,同上
3.创建babel配置文件.babelrc,并添加”presets”:[“es2015”];

运行

如果没有创建bable配置文件也可以使用

或者在package.json中scripts添加任务

执行npm run build即可启动babel自动监听

babel-register

babel-reigster模块改写require命令，为它加上一个钩子.伺候每当使用require加载.js,.jsx,.es和.es6后缀名的文件,就会先用Babel进行转码.(实时转码,所以只适合在开发环境使用)
使用时,必须首先加载babel-register//require(“babel-register”).

babel-polyfill:

Babel默认只转换新的JavaScript句法,而不转换新的API,比如Iterator,Generator,Set,Maps,Proxy,Reflect,Symbol,Promise等全局对象,以及一些定义在全局对象上的方法(比如Object.assign)

举例:ES6在Array对象上新增了Array.from方法.Babel就不会转码这个方法.如果想让这个方法允许,必须使用babel-polyfill,为当前环境提供一个垫片

使用浏览器版本babel

Babel6.0开始,不再直接提供浏览器版本,需要使用构建工具构建,或者通过5.x版本的babel-core模块获取.
在node_modules/babel-core/browser.js是Babel提供的转换脚本,可以在浏览器允许.ES6脚本放在script标签中,并注明 type=’text/babel’
另一种方法是使用babel-standalone模块提供的浏览器脚本.
网页中实时将ES6代码转为ES5,对性能会有影响.生产环境需要加载已经转码后的脚本

Babel提供一个在线编译器

https://babeljs.io/repl/#?babili=false&evaluate=true&lineWrap=false&presets=es2015%2Creact%2Cstage-2&code=

Google的Traceur转码器，也可以将ES6代码转为ES5代码。
